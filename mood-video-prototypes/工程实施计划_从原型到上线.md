# 心情视频 · 从原型到上线 — 完整工程实施计划

> 本文档回答「原型做完了，接下来需要做什么才能让用户真正用起来」。
> 覆盖从技术选型、团队搭建、功能开发、测试、合规到上线的全流程。

---

## 总览：6 个阶段，约 10-12 个月

```
原型验证 ✓ → Phase 1: 地基 (2个月) → Phase 2: MVP (3个月) → Phase 3: 双端+合规 (2个月) → Phase 4: 上线冲刺 (1个月) → Phase 5: 运营迭代 (持续)
```

## Phase 1：地基工程（第 1-2 个月）

> **目标：技术选型落地、CI/CD 就绪、第一个 API 跑通**

### 1.1 技术栈最终定案

| 决策项 | 推荐方案 | 理由 |
|-------|---------|------|
| **前端框架** | Flutter 3.x | iOS+Android 一套代码，老年人市场 Android 占比高 |
| **后端语言** | Node.js (NestJS) 或 Go (Gin) | NestJS 招人快；Go 性能好适合视频处理 |
| **数据库** | PostgreSQL 15 + Redis | PG 存结构化数据，Redis 缓存+消息队列 |
| **对象存储** | 腾讯云 COS + CDN | 视频分发全国覆盖，腾讯生态整合好 |
| **消息队列** | RabbitMQ / 腾讯云 CMQ | AI 视频生成是异步任务，必须解耦 |
| **地图服务** | 腾讯地图 SDK + WebService API | 定位、围栏、逆地理编码 |
| **推送服务** | 腾讯信鸽 / 极光推送 | 子女端告警推送 |
| **即时通讯** | 腾讯云 IM | 老友圈聊天功能 |
| **日志监控** | 腾讯云 CLS + Sentry | 错误追踪 + 性能监控 |

### 1.3 数据库 Schema 初版

```

## Phase 2：MVP 开发（第 3-5 个月）

> **目标：一句话→视频的核心闭环跑通，可内测**

### 2.1 AI 视频生成管线（最核心，第 3-4 个月）

这是整个产品的心脏。需要打通：

```
用户输入（文字/语音）
  → 情绪分析（LLM）
  → 画面描述生成（LLM）
  → 视频生成（AI 视频模型 API）
  → 字幕叠加（FFmpeg）
  → 背景音乐匹配
  → 视频合成输出
  → 上传 COS + 返回播放链接
```

**具体任务分解：**

| 周 | 任务 | 交付物 |
|----|------|-------|
| W1-2 | 对接 AI 服务商 API（即梦/可灵/通义千问），获取 SDK 权限 | API Key + 接口文档 + 封装好的 Service |
| W3-4 | 情绪分析服务：输入文本 → 输出情绪标签+置信度 | EmotionService |
| W5-6 | 画面描述生成：情绪+原始文本 → AI 画面 prompt | PromptGenerator |
| W7-8 | 视频生成调用：prompt → 视频 URL | VideoGenService |
| W9-10 | 视频后处理：FFmpeg 叠加字幕+音乐+片头片尾 | VideoProcessor |
| W11-12 | 异步任务队列：提交任务 → 轮询状态 → 完成回调 | TaskQueue + Webhook |

**AI 服务商评估（建议在 W1-2 做 POC）：**

| 服务商 | 能力 | 价格参考 | 延迟 | 备注 |
|-------|------|---------|------|------|
| 即梦 AI | 文生视频 | ~0.3元/次 | 30-90s | 字节系，效果好 |
| 可灵 AI | 文生视频 | ~0.5元/次 | 60-120s | 快手系，风格多样 |
| 通义千问 | 文本理解 | ~0.001元/千token | <1s | 情绪分析用 |
| 讯飞语音 | 语音转文字 | ~0.01元/次 | <2s | 支持方言 |
| FFmpeg | 视频合成 | 免费 | 视视频长度 | 服务端部署 |

### 2.3 后端 API 开发（与 APP 同步）

**核心 API 列表：**

```
POST   /api/auth/login            — 手机号验证码登录
POST   /api/auth/send-code        — 发送验证码
GET    /api/user/profile          — 获取用户信息

POST   /api/video/create          — 提交视频生成任务（文字/照片/语音）
GET    /api/video/status/:taskId  — 查询生成进度
GET    /api/video/list            — 我的视频列表
GET    /api/video/:id             — 视频详情+播放链接
DELETE /api/video/:id             — 删除视频

POST   /api/upload/image          — 上传图片（返回 COS URL）
POST   /api/upload/video          — 上传视频片段（返回 COS URL）

POST   /api/health/checkin        — 健康打卡
GET    /api/health/records        — 打卡记录
GET    /api/health/weekly-report  — 健康周报

GET    /api/share/wechat-sign     — 微信分享签名

### 3.5 合规审查（第 7 个月，贯穿全程）

这是最容易忽视但最致命的一环，单独列出：

| 合规项 | 具体要求 | 实现方式 |
|-------|---------|---------|
| **个人信息保护法** | 收集个人信息需明示同意 | APP 首次启动弹出隐私政策 + 用户协议，必须点「同意」才能用 |
| **敏感个人信息** | 健康数据、位置数据属于敏感信息，需单独授权 | 每个功能模块（健康打卡、GPS 定位）首次使用时弹出单独授权弹窗 |
| **数据最小化** | 只收集必要数据 | 位置仅存 30 天，视频 1 年未查看自动归档 |
| **数据可携带** | 用户可导出自己的数据 | 设置页提供「导出我的数据」按钮 |
| **账号可注销** | 用户可彻底删除账号 | 设置页提供「注销账号」，7 天冷静期后彻底删除 |
| **未成年人保护** | 需确认用户年龄 | 注册时填写年龄，< 18 岁进入未成年人模式 |
| **App 备案** | 国内 APP 必须完成 ICP 备案 + App 备案 | 上线前 2 个月启动 |
| **医疗器械资质** | 健康数据展示不等于医疗诊断 | 在健康页面显著位置注明"仅为健康参考，不构成医疗建议" |
| **地图合规** | 使用合规地图服务商 | 使用腾讯地图（已通过自然资源部审核） |
| **算法备案** | AI 生成内容需算法备案 | 在网信办完成算法备案 |

### 3.7 测试策略（贯穿全程）

**功能测试：**
- [ ] 核心流程回归测试（每次发版必跑）
- [ ] 异常场景测试（网络断开、上传失败、AI 超时）
- [ ] 老年用户可用性测试（找 5-10 位老人真机测试，记录操作困难点）

**性能测试：**
- [ ] 视频生成并发压力测试（模拟 100/500/1000 并发任务）
- [ ] API 响应时间基准测试（P99 < 500ms）
- [ ] GPS 上报吞吐量测试

**安全测试：**
- [ ] API 渗透测试（SQL 注入、XSS、CSRF）
- [ ] 数据加密验证（传输 TLS 1.3、存储 AES-256）
- [ ] 权限越权测试（A 用户能否访问 B 用户的健康数据）

**合规测试：**
- [ ] 隐私政策弹窗是否正常展示
- [ ] 敏感权限授权流程是否正确
- [ ] 数据导出和注销功能是否可用
